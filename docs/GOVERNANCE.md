# Governance

NativeFrame UI uses a maintainer-led, review-first model while the project is establishing its contributor community.

- Maintainers protect the pure Win32 scope, public API consistency, and release quality.
- Contributors propose changes through issues and pull requests.
- At least one maintainer review is required for code, build, public API, or resource changes.
- Breaking API changes require a roadmap entry and migration guidance.
- The default branch must remain buildable and pass the documented CMake/CTest path.

As the contributor base grows, maintainers may add area owners and a documented decision process without changing the MIT license or the project scope by default.
