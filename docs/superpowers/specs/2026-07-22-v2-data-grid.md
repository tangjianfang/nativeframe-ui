## Goal
Provide a tabular data grid with virtualized rows, column sorting,
multi-column headers, cell editing, and selection models. The V1
ListView is single-column-friendly; the data grid is for many columns
and many rows.

## Why deferred from V1
- Virtualization (only render visible rows) requires careful
  scrollbar / viewport math.
- Cell editing needs inline Control hosts per cell type.
- Sorting across columns requires per-column comparators and
  type-aware promotion.

## Architecture
- `nfui::DataGridView` (top-level Control).
- `nfui::DataGridColumn` (header + width + sort direction + cell
  template type).
- `nfui::DataGridRow` (virtual — only realized when in viewport).
- `nfui::DataGridCellEditor` (per-type editor factories).
- `nfui::DataGridModel` (data source interface).

## Components
- New `nfui_datagrid` library.
- `nfui::DataGridViewCreateParams` (column defs, model, selection mode).

## Open questions
- Editable vs read-only cells: per-column flag or per-cell?
- Filtering: separate control, or built-in header dropdown?
- Frozen columns (always-visible left/right)?
