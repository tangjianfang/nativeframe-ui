## Goal
Allow panes (toolboxes, property grids, file explorers, document
windows) to be docked to the edges of a main window, floated as
undocked windows, or auto-hidden as sidebars. Built on top of
existing `nfui::Splitter` and `nfui::Layout`.

## Why deferred from V1
- Splitter does horizontal/vertical splits; docking needs N-pane layout
  with arbitrary nesting.
- Drag-to-dock needs accurate drag preview (semi-transparent window).
- Tabbed grouping of multiple panes is a UI paradigm shift requiring
  state management beyond V1's `WorkbenchState`.

## Architecture
```
DockSite (a Window)
  ├── DockPane (a Window)
  │     ├── TabStrip (a Control)
  │     └── Content (a Control)
  ├── DockEdge [north|south|east|west]
  └── AutoHideBar
```

State persisted via V1's `nfui::WorkbenchState` extended with
`dock_layout` field.

## Components
- `nfui::DockSite` (top-level container).
- `nfui::DockPane` (a tabbed or single-pane container).
- `nfui::DockEdge` (top/left/right/bottom docks).
- `nfui::AutoHideBar` (collapsed-pane chrome).
- `nfui::DockDragController` (drag-to-dock preview logic).

## Open questions
- Drag preview: `SetLayeredWindowAttributes` alpha, or DWM thumbnail?
- Multi-monitor: undocked windows should respect current monitor DPI.
- Saved layouts: how to handle plug-in panes that don't exist across
  versions?
