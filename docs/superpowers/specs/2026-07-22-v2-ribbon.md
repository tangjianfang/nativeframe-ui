## Goal
Provide an Office-style ribbon (tabs → groups → buttons/galleries) as
a top-level control. Replaces traditional menu + toolbar for apps that
want modern command surfaces.

## Why deferred from V1
- 12+ sub-controls (Tab, Group, Button, SplitButton, DropDownGallery,
  ComboBox, Spinner, StatusBar, etc.).
- Contextual tabs (developer ribbon, picture ribbon) need state
  coordination.
- Customization dialog (show/hide tabs, reorder) is its own subsystem.

## Architecture
- `nfui::Ribbon` (top-level Window subclass).
- `nfui::RibbonTab` ("Home", "Insert", etc.).
- `nfui::RibbonGroup` (label + controls).
- `nfui::RibbonButton`, `RibbonSplitButton`, `RibbonGallery`, etc.
- Reuses `nfui::Button`, `nfui::CommandRouter` for command dispatch.

## Components
- New `nfui_ribbon` library (sub-lib under the V2 umbrella).
- `nfui::RibbonCreateParams` (tabs, default-selected, mode).

## Open questions
- Minimize behavior: title bar collapse to a single row of tabs?
- Keyboard navigation: Ctrl+Tab between tabs, arrows within group?
- Touch mode: separate larger-button mode for tablet/convertible?
