## What they have
- Modern (since 2020), Rust core with C++/JS/Python bindings.
- Declarative `.slint` markup language.
- Hardware-accelerated rendering (Skia or Femtovg).
- Cross-platform: Win32, macOS, Linux, embedded, wasm.
- Live preview via SlintPad.

## What we have that they don't
- Pure C++20 with no Rust dependency.
- Native Win32 HWND per Control (not a custom renderer).
- Per-component STATIC library split.
- No markup language — direct C++ API.

## What we explicitly choose NOT to clone
- Declarative markup language (V1 non-goal).
- Custom renderer (V1 uses Win32; hardware acceleration V2 candidate).
- Live preview tooling (V1 non-goal).

## Positioning
Slint is the "modern declarative UI for embedded + desktop" choice.
NativeFrameUI is the "Win32-native C++ imperative UI" choice. Slint
prioritizes cross-platform + declarative; we prioritize Win32 fidelity
+ native message loop + static linking.
